docker

概念

docker是一个应用打包、分发、部署的工具，也可以理解为一个轻量的虚拟机
  • 打包：即是把软件运行所需的依赖、第三方库、软件打包到一起，变成一个安装包
  • 分发：把打包好的安装包上传到一个镜像仓库，可以让其他人非常方便的获取和安装
  • 部署：拿着安装包就可以使用一个命令运行起来应用，自动模拟出一摸一样的运行环境，不限制运行平台

重要的两个概念

  • 镜像：可以理解为软件安装包，方便进行传播和安装
  • 容器：软件安装后的状态，每个软件运行滑行都是独立的、隔离的

安装

桌面端：[https://www.docker.com/products/docker-desktop](https://www.docker.com/products/docker-desktop)
服务器端：[https://docs.docker.com/engine/install/#server](https://docs.docker.com/engine/install/#server)

开启或者安装 Linux 子系统

  • 桌面端安装，需要在控制面板 -> 程序 -> 启用或关闭 Windows 功能 中开启Hyper，如果发现没有，可以通过知乎进行安装，只针对 windows11，当然，windows10 也可以在网上找到对应的方法
  • 命令行安装 Linux 内核：wsl.exe --install -d Ubuntu，也可以通过：wsl --update
  • 设置开机启动 Hypervisor：bcdedit /set hypervisorlaunchtype auto
  • 设置默认使用版本 2：wsl.exe --set-default-version 2，当然，实际发现不用也行，用了反而有时候会出错
  • 注意 PowerShell 要用管理员权限打开
  • 查看 WSL 是否正确安装：wsl.exe --list --verbose
  • 确保电脑已经开启虚拟化，就是 hyper 是否打开了，可以通过任务管理器查看 cpu 中的虚拟化是否显示“已启用”

docker 快速安装软件

可以在docker-desktop的首页的搜索栏直接搜索你所需要的软件，点击安装即可；也可以在docker的官方镜像仓库查找：[docker镜像仓库](https:hub.docker.com/)，然后通过命令行的方式运行，比如：安装redis：`docker run -d -p 6379:6379 --name redis redis:latest`，其中`-d`代表后台运行，`-p`代表将后面的端口号映射到本机的哪个端口上，`--name`后面跟上起的名字，最后那个代表你使用的哪个镜像安装的

命令参考：https://docs.docker.com/engine/reference/commandline/run/

更多相关命令

  • docker ps 查看当前运行中的容器
  • docker images 查看镜像列表
  • docker rm container-id 删除指定 id 的容器
  • docker stop/start container-id 停止/启动指定 id 的容器
  • docker rmi image-id 删除指定 id 的镜像
  • docker volume ls 查看 volume 列表
  • docker network ls 查看网络列表

  • 编写一个 Dockerfile

Build 为镜像（安装包）和运行

  • 编译：docker build -t test:v1 其中-t是设置镜像名字和版本号
  • 运行：docker run -p 8080:8080 --name test-hello test:v1 其中-p表示映射容器内端口到宿主机 --name表示容器名字，最后接的是镜像名字

目录挂载

存在的问题：

  • 使用 docker 运行后，当修改项目代码后不会立即生效，还需要重新build和run，很麻烦
  • 容器里面产生的数据，例如 log 文件、数据库备份文件，容器删除后就丢失了

挂载方式

  • bind mount 直接把宿主机目录映射到容器内，适合挂代码目录和配置文件，可挂到多个容器上。该方式使用绝对路径方式 -v 代码存储路径:/app 表示挂载到 /app上
  • volume 由容器创建和管理，创建在宿主机上，所以删除容器不会丢失，官方推荐，更加高效，使用的是 Linux 文件系统，适合存储数据库数据，可挂到多个容器上。该方式只需要一个名字 -v db-data:/app
  • tmpfs mount 适合存储临时文件，存储在宿主机内存中，不可多容器共享
例如：`docker run -p 8080:8080 --name test-hello -v 代码路径:/app -d test:v1`。这就相当于把本地的代码挂载到了容器中的app目录上了，访问还是直接`localhost:8080 + 定义的路径` 即可

注意！因为挂载后，容器里的代码就会替换为你本机的代码了，如果你代码目录没有 node_modules 目录，你需要在代码目录执行下 npm install –registry=https://registry.npm.taobao.org确保依赖库都已经安装，否则可能会提示“Error: Cannot find module ‘koa’” 如果你的电脑没有安装 nodejs，你需要安装一下才能执行上面的命令。

docker-compose

上述我们已经可以进行多容器通信，但是如果项目使用的第三方软件过多，那我们就需要管理很多容器，每个也都需要单独配置运行、指定网络等。为了简化操作，所以需要docker-compose，它可以将项目的多个服务集合到一起，一键运行
  • 如果是安装的桌面端 docker，即 docker-desktop，那么就不需要单独安装 docker-compose，里面已经包括了
  • 如果是没图形界面的服务器版 docker，就需要单独安装 docker compose
  • 可以通过 docker-compose 检查是否安装成功

要把项目依赖的多个服务集合到一起，需要编写一个 dcoker-compose.yml 的文件，描述依赖哪些服务，参考文档

在docker-compose.yml 文件所在目录，执行：docker-compose up就可以跑起来了
  • 在后台运行只需要加一个 -d 参数 docker-compose up -d
  • 查看运行状态：docker-compose ps
  • 停止运行：docker-compose stop
  • 重启：docker-compose restart
  • 重启单个服务：docker-compose restart service-name
  • 进入容器命令行：docker-compose exec service-name sh
  • 查看容器运行 log：docker-compose logs [service-name]

发布和部署

镜像仓库是用来存储build出来的“软件包”的，docker官方提供了一个[镜像库](https://hub.docker.com/)，里面包含了大量镜像，我们也可以把自己的项目进行build，然后进行上传，可以方便传播使用
  • 首先需要在官网上注册一个账号
  • 创建一个镜像库
  • 命令行登录账号 docker login -u username
  • 新建一个 tag，名字必须跟你注册账号一样 docker tag test:v1 username/test:v1
  • 上传 docker push username/test:v1
  • 部署测试是否成功 docker run -dp 8080:8080 username/test:v1

成功后在 docker-compose 中也可以直接使用这个镜像了

由于docker官网的镜像托管上传下载很慢，多以可以使用比如阿里云的免费镜像托管

备份和迁移数据

容器中的数据，如果没有用挂载目录，删除容器后就会丢失数据。前面我们已经讲解了如何 挂载目录。如果你是用bind mount直接把宿主机的目录挂进去容器，那迁移数据很方便，直接复制目录就好了。如果你是用volume方式挂载的，由于数据是由容器创建和管理的，需要用特殊的方式把数据弄出来

备份

  • 运行一个 ubuntu 的容器，挂载需要备份的 volume 到容器，并且挂载宿主即目录到容器里的备份目录
  • 运行 tar 命令把数据压缩为一个文件
  • 把备份文件复制到需要导入的机器

导入

  • 运行 ubuntu 容器，挂载容器的 volume，并且挂载宿主机备份文件所在目录到容器里
  • 运行 tar 命令解压备份文件到指定目录

备份 MongoDB 数据演示

  • 运行一个 mongodb，创建一个名叫 mongo-data 的 volume 指向容器的 /data 目录：docker run -p 27017:27017 --name mongo -v mongo-data:data -d mogo:4.4
  • 运行一个 Ubuntu 的容器，挂载 mongo 容器的所有 volume，映射宿主机的 backup 目录到容器里面的 /backup 目录，然后运行 tar 命令把数据压缩打包：docker run -rm --volumes-from mongo -v d:backup:/backup ubuntu tar cvf /backup/backup.tar /data/ 之后就可以使用这个 backup.tar 文件导入到其他地方或者设备了。其中的 --rm表示如果已经存在同名容器就先删除

恢复 Volume 数据演示

  • 运行一个 ubuntu 容器，挂载 mongo 容器的所有 volumes，然后读取 /backup 目录中的备份文件，解压到 /data/ 目录：docker run -rm --volumes-from mongo -v d:/backup/backup ubuntu bash -c "cd /data/ && tar xvf /backup/backup.tar --strip 1
**注意，volumes-from 指定的是容器名字；strip 1 表示解压时去掉前面1层目录，因为压缩时包含了绝对路径**
